gitlab数据备份与恢复

gitlab定时自动备份

0 2 * * * /opt/gitlab/bin/gitlab-rake gitlab:backup:create

gitlab修改备份路径： 修改/etc/gitlab/gitlab.rb文件

gitlab_rails['backup_path'] = '/mnt/backups'

gitlab恢复

1. 停止相关数据连接服务

gitlab-ctl stop unicorn
gitlab-ctl stop sidekiq

2. 从1393513186编号备份中恢复

gitlab-rake gitlab:backup:restore BACKUP=1393513186

如果备份文件没有多个，应该是不用指定名称的。

2.1. 启动Gitlab

sudo gitlab-ctl start

2.2. Gitlab迁移

把备份文件拷贝到gitlab的备份目录下，根据上面gitlab恢复步骤即可。

3. 注意事项

  1. Gitlab的备份，必须在相同版本下使用才能恢复。
  2. 备份恢复后，有一个bug，所以备份的时候，一定注意多备份两个文件：
    1. /etc/gitlab/gitlab-secrets.json
    2. /etc/gitlab/gitlab.rb 参考：(https://gitlab.com/gitlab-org/gitlab-ce/issues/17873) 其中有说明gitlab-ce和非CE版本需要执行的修复脚本。
  3. 如果实在无法恢复2.1中的文件，可以通过执行如下指令，此指令验证在8.9.6版本是可以执行，但更高的8.12则无法成功执行：
    gitlab-rails runner "Project.where.not(import_url: nil).each { |p| p.import_data.destroy if p.import_data }"
